# NetHunter Kernel v1.0.0 — Nothing Phone (1) (spacewar)

First public release.

## Quick Install

→ Read [INSTALL.md](https://github.com/ilyamen/nothing-phone-1-nethunter-kernel/blob/master/docs/INSTALL.md)

```bash
# 1. Patch our boot.img through Magisk app, then flash via fastboot
adb push spacewar-nethunter-kernel-v1.0.0-boot.img /sdcard/Download/
# Open Magisk → Install → Patch a File → select our boot.img
adb pull /sdcard/Download/magisk_patched-XXX.img
adb reboot bootloader
fastboot flash boot_a magisk_patched-XXX.img
fastboot reboot

# 2. Install Magisk modules (Magisk Manager → Modules → Install from storage)
#    REQUIRED: nh-overlay-base.zip, nh-logcatd.zip
#    OPTIONAL: pick what you need from the rest

# 3. Reboot — done
```

**Phone preparation (one-time, before this kernel):**
- Install LineageOS 23.2 ([wiki.lineageos.org/devices/spacewar/install](https://wiki.lineageos.org/devices/spacewar/install))
- Install Magisk v30+ ([github.com/topjohnwu/Magisk/releases](https://github.com/topjohnwu/Magisk/releases))

## What's in this release

**Kernel:**
- LineageOS 23.2 base + Kali NetHunter QCACLD-3.0 frame injection (14 patches)
- Linux 5.4.302-qgki, AOSP Clang r547379
- CFI fix for 3 Realtek USB Wi-Fi out-of-tree drivers
- Full NetHunter feature set: USBIP, BT USB Arsenal, MAC80211_HWSIM, HID Gadget (BadUSB-ready), Mass-Storage Gadget (DriveDroid), USB Serial drivers (FTDI/CH341/PL2303/CP210x + cellular), USB Ethernet (RTL8152/AX88179) + USB-Eth gadget (RNDIS/ECM/EEM), DVB-USB-RTL28xxU (RTL-SDR), WireGuard, NTFS+CIFS, L2TPv3+PPP_MULTILINK, TC actions, NFS V3/V4

**9 Magisk modules:**
- `nh-overlay-base` v1.1 — NetHunter app + Terminal + F-Droid client + F-Droid Privileged Extension in /system/priv-app. **Auto-adds NetHunter Store + IzzyOnDroid repos on first PIN unlock** via `fdroidrepos://` intents (one-time tap "Add" on each F-Droid prompt — no manual URL entry). Replaces the deprecated NetHunter Store with a maintained F-Droid client.
- `nh-logcatd` v2 — persistent logcat (64MB rotating) + hourly dmesg + auto-freeze panic incidents (preserves up to 5 panic snapshots forever)
- `realtek-wifi-cfi-fix` — auto-loads 3 Realtek USB Wi-Fi drivers
- `nh-wifi-adb` — ADB on TCP port 44444
- `nh-batch5-vpn` — WireGuard + crypto deps
- `nh-batch5-storage` — NTFS, CIFS modules
- `nh-batch5-net-extras` — USB CDC, PPP, L2TP modules
- `nh-batch5-tc` — TC actions modules
- `nh-batch6-usb-serial` — USB Serial + cellular USB modems

**Verified working:**
- pstore-based kernel panic capture (verified via sysrq-trigger panic test)
- F-Droid Privileged Extension silent-install capability (INSTALL_PACKAGES granted=true)
- 80+ kernel modules loaded across vendor + NetHunter Magisk modules
- All 3 Realtek USB Wi-Fi adapters work in monitor mode + injection
- Internal Wi-Fi (Qualcomm WCN6855): frame injection works in managed mode (verified with `aireplay-ng` deauth while associated to AP). **Full monitor mode switch via `iw dev wlan0 set type monitor` returns ENOTSUPP (-95) — firmware-level limitation despite `iw phy info` listing `monitor` as supported. Use external USB adapter for monitor capture.**

## Known limitations

- LineageOS 23.2 user-builds set `ro.control_privapp_permissions=enforce`. Our nh-overlay-base ships `privapp-permissions-nh-overlay-base.xml` derived from observed system_server crash logs. Future NetHunter app updates that add new privileged permissions will need this XML extended.
- F-Droid repo auto-add fires intents on first PIN-unlock (waits up to 150s for FBE unlock after boot). Two F-Droid prompts ("Add NetHunter Store?" / "Add IzzyOnDroid?") still require one user tap each — F-Droid does not auto-trust new repos by design. Marker `/data/adb/.nh-overlay-base.repos-added` is created after firing; to retry, delete the marker (`su -c rm`) and reboot.
- USB Gadget RNDIS/ECM (USB-Ethernet attack vector against connected hosts) works in parallel with Qualcomm USB_F_GSI but real-world MITM scenarios untested in field.
- Kali chroot install (~1.5 GB rootfs download) is via NetHunter app's Chroot Manager, not bundled in this release.
- CAN bus (car hacking) **cannot** be enabled on this kernel — `CONFIG_CAN` adds a field to `struct net` that breaks GKI vendor module ABI on LineageOS 23.2. If you need CAN, build it as an out-of-tree `.ko`.
