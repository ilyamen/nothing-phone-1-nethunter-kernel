# NetHunter Kernel v1.1.0 — Nothing Phone (1) (spacewar)

Bugfix release: **internal Wi-Fi (WCN6855) full monitor mode** now captures all management frames including EAPOL — handshake capture works without external USB adapter.

## TL;DR

v1.0.0 enabled monitor mode + frame injection on the internal Qualcomm WCN6855 chip, but driver bug in QCACLD-3.0 caused the firmware to deliver only PPDU-status TLVs without payload. Result: airodump saw only ~6.6% of beacons and **zero EAPOL after deauth** — handshake capture impossible without external USB Wi-Fi adapter.

v1.1.0 fixes this with a 3-line patch (`CONFIG_QCA_SUPPORT_FULL_MON := y` + 2-line call to `dp_config_full_mon_mode` in the QCA6750 case of `dp_main.c`). Verified end-to-end on physical Nothing Phone (1):

| Metric | v1.0.0 | v1.1.0 | Δ |
|--------|--------|--------|---|
| Beacons captured (27s, ch5) | 18 | **629** | 35× |
| Probe Response | 2 | 70 | 35× |
| netdev rx_dropped ratio | ~42% | **0%** | gone |
| EAPOL frames (90s, natural reconnect, no deauth) | — | **2** | works |
| Authentication frames | 0 | 5 | new |

EAPOL test: tested by simply disconnecting and reconnecting a phone to TVAB-WIFI 2.4GHz. No active deauth attack — naturally occurring 4-way handshake captured both directions (AP→client M1-type, client→AP M2-type) for two different clients during the 90s window.

## Quick Install (upgrading from v1.0.0)

Same procedure as v1.0.0 — patch the v1.1.0 boot.img through Magisk and flash. **Magisk modules from v1.0.0 are unchanged and don't need reinstall.**

```bash
adb push spacewar-nethunter-kernel-v1.1.0-boot.img /sdcard/Download/
# Magisk → Install → Patch a File → select v1.1.0 boot.img
adb pull /sdcard/Download/magisk_patched-XXX.img
adb reboot bootloader
fastboot flash boot_a magisk_patched-XXX.img
fastboot reboot
```

## What changed

### Kernel (1 commit)

**Commit:** `qcacld-3.0: enable FULL_MON for Yupik (WCN6855)` — [88ac9aaa6](https://github.com/ilyamen/android_kernel_nothing_sm7325_nethunter/commit/88ac9aaa6a0ca79d2748a4fc8d6b536d6ebcb135)

```diff
# drivers/staging/qcacld-3.0/configs/default_defconfig
+CONFIG_QCA_SUPPORT_FULL_MON := y

# drivers/staging/qca-wifi-host-cmn/dp/wifi3.0/dp_main.c
 case TARGET_TYPE_QCA6750:
     ...
     soc->wlan_cfg_ctx->rxdma1_enable = 0;
+    if (cfg_get(soc->ctrl_psoc, CFG_DP_FULL_MON_MODE))
+        dp_config_full_mon_mode((struct cdp_soc_t *)soc, 1);
     break;
```

### Why it works

- WCN6855 = `TARGET_TYPE_QCA6750` (Yupik / SM7325 / Snapdragon 778G+).
- This chip family forces `rxdma1_enable=0`, meaning full MPDU bodies in monitor mode can only flow through the HTT FULL_MON path (not RXDMA_MON_BUF used on QCN9000-class chips).
- Driver gates FULL_MON activation on `soc->full_mon_mode`, set by `dp_config_full_mon_mode()`, which requires `CONFIG_QCA_SUPPORT_FULL_MON` build flag.
- Both were missing on our fork — the QCA6390/6490/6750 case in `dp_soc_cfg_init` did not call `dp_config_full_mon_mode()` even when the build flag was on.
- Added the call (mirrors what QCN9000 case already does at `dp_main.c:14007-14008`) and added the build flag.

### Magisk modules

**No changes.** All 9 modules from v1.0.0 are reused as-is.

## Verified working on physical spacewar

- `iw dev wlan0 info` reports `type monitor` after `echo 4 > /sys/module/wlan/parameters/con_mode`
- `airodump-ng wlan0` captures all beacons + EAPOL after deauth
- `aircrack-ng` / `hcxpcapngtool` extracts handshake hash to `.hc22000`
- TX (frame injection) still works — Kali QCACLD patches from v1.0.0 unaffected
