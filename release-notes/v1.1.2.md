# NetHunter Kernel v1.1.2 — Nothing Phone (1) (spacewar)

Bugfix release — closes 4 issues found in v1.1.1 post-deployment review.

Same kernel commit as v1.1.0/v1.1.1 (`88ac9aaa6` — FULL_MON internal Wi-Fi monitor RX). Kernel ABI unchanged. Drop-in upgrade.

## What's fixed

| # | Bug | Fix |
|---|-----|-----|
| 1 | Bootloader's `log_buf_len=256K` overrode our `CONFIG_LOG_BUF_SHIFT=19` (512K) → kernel actually used 256K printk ring instead of 512K. | Append `log_buf_len=524288` to our CMDLINE in `scripts/04-configure.sh` so it overrides bootloader. Verified `cat /proc/cmdline | grep log_buf_len` after flash → `log_buf_len=524288`. |
| 2 | `nh-logcatd` panic-incident filter triggered on `console-ramoops` (which is written every boot, not just on panic) → 4 false-positive "panic" incidents per day of normal use. | Filter narrowed to **only** `dmesg-ramoops` (real oops marker). `console-ramoops` and `pmsg-ramoops` continue to be archived but no longer flag an incident. |
| 3 | `magisk --zygisk` CLI option doesn't exist in Magisk 30.7 → `nh-collect-logs.sh` errored on every run. | Removed the `--zygisk` line. |
| 4 | SELinux AVC watcher captured 200 spam denials per day from `comm="ip"` shell→shell `netlink_route_socket` — these come from manual `ip link …` runs in adb shell, not module issues. | Filter excludes `scontext=u:r:shell:s0 tcontext=u:r:shell:s0 tclass=netlink_route_socket`. Real module/system denials still captured. |

## Quick Install (upgrading from v1.1.1)

```bash
adb push spacewar-nethunter-kernel-v1.1.2-boot.img /sdcard/Download/
# Magisk → Install → Patch a File → select v1.1.2 boot.img
adb pull /sdcard/Download/magisk_patched-XXX.img
adb reboot bootloader
fastboot flash boot_a magisk_patched-XXX.img    # (or boot_b — your active slot)
fastboot reboot

# Then in Android, install the updated nh-logcatd (Magisk → Modules → Install zip):
adb push nh-logcatd-v1.1.2.zip /sdcard/Download/
```

## Compatibility

- Same kernel ABI as v1.1.0/v1.1.1 — drop-in upgrade.
- Other Magisk modules (`nh-overlay-base`, `nh-batch5-*`, etc.) — unchanged.

## Known limitations (unchanged from v1.1.1)

Heavier debug detectors (`DETECT_HUNG_TASK`, `LOCKDEP`, `KASAN`) cannot be enabled without rebuilding `qca_cld3_qca6750.ko` and other vendor `.ko` files from source. See `docs/ROADMAP.md` "ABI fragility lessons" section for the full table of which kernel debug flags break vs work on this device.
